## Path: ai/hyperpredict_native.py
## LOC: ~1,200 | Priority: MEDIUM | Tier: cognition

**Purpose:** Massively parallel time-series forecasting. Maintain up to 10^6 independent forecast horizons (one per tracked entity) with low per-series memory. Ensemble of lightweight models, online-updated.

**Public API:**
```python
@dataclass
class Forecast:
    series_id: str; horizons: list[float]; intervals: list[tuple[float,float]]    # confidence intervals

class Hyperpredict:
    def observe(self, series_id: str, t: float, value: float) -> None: ...
    def predict(self, series_id: str, steps: int) -> Forecast: ...
    def batch_predict(self, series_ids: list[str], steps: int) -> list[Forecast]: ...
    def ensemble_weights(self, series_id: str) -> dict[str,float]: ...
```

**Internal Architecture:**
- Per-series state — compact: last N points (ring buffer), EWMA params, ARIMA-lite (p=1,d=1,q=1) state, decomposition (trend/season/residual via simple regression)
- Ensemble — weighted average with online weight learning (exponentiated gradient on prediction error)
- Batch processing — vectorized across series via array.array packed storage

**Key Algorithms:**
- Exponential weighted moving average
- ARIMA(1,1,1) — closed-form OLS
- Seasonal decomposition (STL-lite)

**Acceptance Tests:**
1. 10,000 synthetic sine + noise series → average MAPE < 5% at 10-step horizon
2. Ensemble weights converge to favor best model on each series within 100 observations
3. Memory footprint ≤ 1KB per series (proven via sys.getsizeof traversal)
