# Task Brief: Multi-Repo Integration Batch
## Tugas: Integrasi 5 Repo ke MAGNATRIX

### Repos:
1. **browser-use/browser-use** — AI control browser via Playwright
2. **microsoft/playwright-python** — Web automation SDK
3. **huggingface/ml-intern** — Auto ML Engineer (300 iteration agentic loop)
4. **julep-ai/julep** — Multi-step agent dengan memory
5. **Gitlawb** (org) — Decentralized code collaboration platform:
   - `openclaude` (27.4k stars) — "runs anywhere. uses anything"
   - `agentvm` — Multi-agent bash CLI dengan tmux sandbox
   - `node` — Decentralized git node (Rust, libp2p gossip, Ed25519 identity)
   - `banker-skills` — Plug-and-play agent skills

### Deliverables:
Untuk setiap repo, buat:
1. Bridge adapter file di `collective-brain/adapters/` atau layer yang sesuai
2. Skill template di `skills/{repo-name}/SKILL.md`
3. Update `.env.example` dengan config baru (jika ada)

### Mapping ke Layer MAGNATRIX:
- browser-use → Layer 7 (Browser)
- playwright → Layer 7 (Browser automation skill)
- ml-intern → Layer 10 (Uncensored AI / Auto ML)
- julep → Layer 6 (Skills / Agent memory)
- Gitlawb/node → Layer 4 (P2P Mesh) — decentralized node
- Gitlawb/agentvm → Layer 6 (Agent execution / sandbox)
- Gitlawb/openclaude → Layer 1.5 (Alternative LLM router)
- Gitlawb/banker-skills → Layer 6 (Skill ecosystem integration)

### Acceptance Criteria:
- Semua file Python harus pass syntax check
- Bridge harus punya demo `if __name__ == "__main__"`
- Skill harus mengikuti format SKILL.md yang sudah ada
- File harus di `/mnt/agents/MAGNATRIX-OS/`

### Format File Name:
- Bridge: `{repo_short}_bridge.py`
- Skill dir: `skills/{skill-name}/`
