# Module 8: Long-Term Episodic Memory
## Path: knowledge/episodic_native.py
## LOC: ~1,100 | Priority: HIGH | Tier: memory

**Purpose:** Lossless append-only replay buffer of every (timestamp, agent, observation, action, reward, embedding) tuple. Periodic "sleep-cycle" consolidation clusters episodes into semantic patterns stored in the graph DB.

**Public API:**
```python
@dataclass
class Episode:
    id: str; ts: float; agent_id: str; observation: dict; action: Optional[Action]; reward: Optional[float]; embedding: list[float]; tags: list[str]

class EpisodicMemory:
    async def write(self, ep: Episode) -> None: ...
    async def query(self, q: str, k: int = 10) -> list[Episode]: ...    # semantic + temporal
    async def consolidate(self) -> int: ...   # returns # of new semantic clusters
    async def replay(self, agent_id: str, window: tuple[float,float]) -> Iterator[Episode]: ...
    async def forget(self, criteria: dict) -> int: ...    # explicit forgetting only
```

**Internal Architecture:**
- Append-only log — SQLite WAL mode, one row per episode
- Hash embedding — deterministic hash-based (reuse existing pattern)
- Consolidation — K-means (hand-rolled) over recent N episodes, extract centroids, write to graph DB as semantic nodes
- Compaction — old episodes (>30 days) get summarized; raw kept compressed (zlib)

**Key Algorithms:**
- K-means with k-means++ init
- Streaming reservoir sampling for consolidation window

**Acceptance Tests:**
1. Write 10,000 episodes; query by content returns top-k with recall ≥ 0.9
2. Consolidation reduces 1000 similar episodes to ≤ 50 clusters
3. Replay returns episodes in strict temporal order
4. Forget(criteria) removes only matching episodes
5. Restart preserves all episodes
