# ASI Expansion Phase 2 — 7 Modules Brief

## Module 4: Causal Reasoning Core
**Path:** ai/causal_reasoning_native.py | **LOC:** ~900 | **Priority:** HIGH

**Purpose:** Pearl-style causal inference. Structural Causal Models (SCMs) with do-calculus for intervention reasoning, counterfactual inference, and confounder detection.

**Public API:**
- `CausalGraph` — DAG with nodes (variables), edges (causal), and do-interventions
- `CausalGraph.do(node, value)` — returns interventional distribution
- `CausalGraph.counterfactual(evidence, intervention)` — "what if X had been different"
- `CausalGraph.identify(effect, cause)` — is causal effect identifiable from observational data?
- `backdoor_criterion(graph, treatment, outcome)` — check if set Z satisfies backdoor
- `frontdoor_criterion(graph, treatment, outcome)` — check if frontdoor path exists

**Key Algorithms:**
- DAG cycle detection (topological sort)
- Do-calculus: rule 1 (observation insertion), rule 2 (action/observation exchange), rule 3 (action deletion)
- Simple confounder detection via common causes
- Pearl's three-step counterfactual: abduction (infer exogenous), action (intervene), prediction (propagate)

**Acceptance Tests:**
1. Chain X→Y→Z: do(Y=1) changes Z but not X
2. Fork X←Z→Y: backdoor criterion identifies Z as confounder
3. Collider X→Z←Y: conditioning on Z creates spurious correlation
4. Counterfactual: "Smoking patient who got cancer — what if they hadn't smoked?"

## Module 2: Meta-Cognition Layer
**Path:** ai/meta_cognition_native.py | **LOC:** ~800 | **Priority:** HIGH

**Purpose:** Self-model of own reasoning. Confidence calibration, strategy selection, and resource allocation for thinking.

**Public API:**
- `MetaCognitionEngine` — monitors reasoning traces and adjusts strategy
- `confidence(question, answer)` — calibrated probability of correctness
- `select_strategy(task_type)` — choose between search, deduction, analogy, simulation
- `should_stop(thinking_trace)` — halting criterion for reasoning loops
- `reflect_on_error(expected, actual)` — update self-model

**Key Algorithms:**
- Confidence calibration via past accuracy tracking
- Strategy selection bandit (epsilon-greedy over strategy success rates)
- Halting: diminishing returns on thinking time

**Acceptance Tests:**
1. High-confidence predictions are correct ≥80% of time
2. Strategy selection improves with experience
3. Halting stops before overthinking

## Module 7: Counterfactual Engine
**Path:** ai/counterfactual_native.py | **LOC:** ~800 | **Priority:** HIGH

**Purpose:** What-if exploration and regret minimization. Generate alternative histories, evaluate counterfactual utility.

**Public API:**
- `CounterfactualEngine` — explore alternative action sequences
- `what_if(history, alternative_action, step)` — simulate from branch point
- `regret(actions, outcomes, alternative_outcomes)` — compute regret
- `best_alternative(history, available_actions)` — action with lowest regret

**Key Algorithms:**
- Branch simulation using World snapshot/restore
- Cumulative regret computation
- Minimax over counterfactual trees

**Acceptance Tests:**
1. What-if correctly changes outcome at branch point
2. Regret higher for obviously-bad decisions
3. Best alternative avoids known bad paths

## Module 5: Theory of Mind
**Path:** ai/theory_of_mind_native.py | **LOC:** ~800 | **Priority:** HIGH

**Purpose:** Model other agents' beliefs, desires, intentions. Recursive belief modeling (I believe that you believe that...).

**Public API:**
- `AgentModel` — model of another agent's mental state
- `predict_action(agent_model, situation)` — predict what agent will do
- `belief(agent_model, proposition)` — agent's confidence in proposition
- `intention(agent_model)` — agent's current goal
- `recursive_belief(depth)` — nth-order theory of mind

**Key Algorithms:**
- Bayesian belief update for agent models
- Recursive belief nesting (bounded depth)
- Intent recognition via goal-plan trees

**Acceptance Tests:**
1. Predicts deception (agent says X but believes not-X)
2. Recursive belief depth ≥ 3 stable
3. Intent recognition from action sequences

## Module 15: Ethical Reasoning Core
**Path:** ai/ethical_reasoning_native.py | **LOC:** ~900 | **Priority:** HIGH

**Purpose:** Multi-framework ethical analysis (deontological, utilitarian, virtue ethics). Evaluate actions against constitutional principles.

**Public API:**
- `EthicalEvaluator` — score actions across frameworks
- `deontological_check(action, rules)` — rule-based permissibility
- `utilitarian_score(action, affected_agents)` — net welfare change
- `virtue_ethics_score(action, virtues)` — alignment with virtues
- `constitutional_check(action, principles)` — MAGNATRIX 14 principles

**Key Algorithms:**
- Rule-based logic for deontology
- Welfare aggregation (sum/average)
- Virtue matching via similarity

**Acceptance Tests:**
1. Harmful actions flagged by all frameworks
2. Beneficial actions approved by all frameworks
3. Dilemmas show framework disagreement
4. Constitutional principles override in conflict

## Module 14: Affective Computing
**Path:** ai/affective_native.py | **LOC:** ~700 | **Priority:** HIGH

**Purpose:** Emotion modeling and empathy simulation. Track emotional state, simulate emotional impact on others.

**Public API:**
- `AffectiveState` — PAD model (Pleasure, Arousal, Dominance)
- `EmotionalReaction(event, personality)` — compute emotional response
- `EmpathySimulator` — infer other's emotional state from context
- `EmotionRegulator` — adjust affective state toward target

**Key Algorithms:**
- PAD emotional space (3D continuous)
- Appraisal theory: event → appraisal → emotion
- Emotional contagion: nearby agents converge in affect

**Acceptance Tests:**
1. Good events → positive pleasure
2. Threats → high arousal + negative pleasure
3. Empathy infers correct emotion from context
4. Regulation reaches target state

## Module 9: Auto-Research Lab
**Path:** knowledge/auto_research_native.py | **LOC:** ~900 | **Priority:** HIGH

**Purpose:** Autonomous scientific discovery loop: hypothesis → experiment → analysis → publish.

**Public API:**
- `ResearchLab` — run autonomous research cycles
- `generate_hypothesis(observations)` — propose testable hypotheses
- `design_experiment(hypothesis)` — create experimental protocol
- `analyze_results(data, hypothesis)` — statistical analysis
- `publish_findings(findings)` — format research output

**Key Algorithms:**
- Pattern induction from observations
- A/B test design with power analysis
- Simple statistical tests (t-test, chi-square)
- Literature synthesis (mock)

**Acceptance Tests:**
1. Hypothesis generation from correlated data
2. Experiment design controls confounders
3. Analysis correctly accepts/rejects hypothesis
4. Full cycle completes without deadlock
