# Task Brief: Untouched Domains Integration (32 modules)

## Context
MAGNATRIX-OS currently has 458 modules. We are adding 4 untouched domains with 8 modules each (32 total), bringing the system to 490 modules.

## Module Architecture Requirements
- Pure Python standard library ONLY (no external dependencies)
- Dataclass-based models
- JSON persistence via pathlib.Path
- Every module must have `to_dict()` and `get_stats()` methods
- `__all__` export list at bottom
- `*_native.py` naming convention
- Self-contained with proper error handling
- All classes must be instantiable with no arguments (default values)

## Domain 3: Vector Database Native (8 modules)
Build a pure-Python vector database subsystem:

1. `vector_hnsw_index_native.py` — Hierarchical Navigable Small World index, multi-layer graph, pure Python
2. `vector_storage_engine_native.py` — Dense vector storage, page allocation, append-only segments
3. `vector_similarity_search_native.py` — Cosine, Euclidean, Manhattan, Hamming distance metrics, top-k search
4. `vector_dimension_reduction_native.py` — PCA-lite, random projection, feature selection for dimensionality
5. `vector_ann_query_engine_native.py` — Approximate nearest neighbor query routing, candidate pool, refinement
6. `vector_index_merger_native.py` — Multi-segment index merge, background compaction, deduplication
7. `vector_batch_inserter_native.py` — Bulk insert, buffering, flush thresholds, id mapping
8. `vector_metadata_filter_native.py` — Metadata indexing, boolean filter queries, filtered vector search

## Domain 4: Real-time WebSocket Pub/Sub (8 modules)
Build a WebSocket pub/sub messaging subsystem (simulated, pure stdlib):

1. `websocket_server_native.py` — WebSocket server simulation, handshake, frame parsing, connection management
2. `websocket_pubsub_broker_native.py` — Publish-subscribe broker, topic routing, message dispatch
3. `websocket_channel_manager_native.py` — Channel creation, deletion, ACL, rate limits, channel listing
4. `websocket_message_queue_native.py` — In-memory message queue, priority queue, TTL, dead letter
5. `websocket_client_registry_native.py` — Client registration, session tracking, connection state, heartbeat
6. `websocket_heartbeat_monitor_native.py` — Ping/pong simulation, timeout detection, disconnection cleanup
7. `websocket_broadcast_engine_native.py` — Fan-out broadcast, room-based messaging, message ordering guarantees
8. `websocket_presence_tracker_native.py` — User presence (online/away/offline), activity tracking, status broadcast
